# dyx-blog 项目需求文档

## 1. 项目背景
`dyx-blog` 的建设目标是打造一个集个人品牌展示、简历表达、博客发布和后台维护于一体的个人站点系统。

项目需要同时满足两类核心诉求：
1. 对外展示：完整呈现个人经历、作品、荣誉、动态与博客内容
2. 对内维护：通过后台高效管理资料、内容、媒体和站点配置

## 2. 项目目标

### 2.1 业务目标
- 提供统一的个人品牌展示入口
- 展示个人资料、教育经历、工作经历和项目经验
- 支持博客与动态内容持续输出
- 支持作品与荣誉的长期沉淀展示
- 提供后台管理能力，降低维护成本
- 支持站点访问统计与运营观察

### 2.2 产品目标
- 页面风格简洁、美观、具有个人特色
- 前台访问体验友好，兼顾桌面端与移动端
- 后台操作路径清晰，适合日常内容维护
- 资料和媒体统一管理，避免分散维护

## 3. 用户角色

### 3.1 游客
游客是公开站点访问者，主要需求包括：
- 浏览首页和个人信息
- 查看关于我、简历、作品和荣誉
- 阅读博客与动态内容
- 下载或查看 PDF 简历
- 留言互动

### 3.2 管理员
管理员是站点维护者，主要需求包括：
- 登录后台
- 管理博客、动态、荣誉、作品、用户
- 维护首页展示内容与个人资料
- 上传和管理媒体资源
- 查看访问统计和访问日志
- 配置系统参数与存储方式

### 3.3 普通后台用户（受限角色）
具备部分内容管理能力，但不一定拥有系统级管理权限。适合日常运营或协助维护人员。

### 4.1 前台范围

- 关于我展示
- 简历展示
- 留言页
- 动态列表与详情
- 博客列表与详情

### 4.2 后台范围

- 仪表盘
- 访问日志
- 博客管理
- 动态管理
- 荣誉管理
- 首页 Hero 管理
- 首页足迹管理
- 关于我管理
- 简历管理

- 媒体管理
- 系统配置
- 用户管理

## 5. 功能需求

### 5.1 登录认证
系统应提供后台登录能力。

功能要求：
- 用户输入用户名和密码完成登录
- 登录成功后返回 JWT Token
- 后台后续请求需自动携带 Token
- Token 失效时前端应自动跳转登录页
- 系统应支持按角色控制后台功能访问权限

### 5.2 首页展示
系统应提供首页聚合展示能力。

功能要求：
- 展示站点标题、主标题、副标题和视觉素材
- 展示最新博客
- 展示最新动态
- 展示精选项目
- 展示最新荣誉
- 展示足迹地图或足迹列表
- 支持访问量上报

### 5.3 关于我展示
系统应展示完整个人信息。

功能要求：
- 展示头像
- 展示个人介绍
- 展示联系方式
- 展示作品列表
- 展示荣誉时间线
- 联系方式支持点击跳转，如邮箱、电话、GitHub

### 5.4 简历展示
系统应提供在线简历展示与 PDF 简历能力。

功能要求：
- 展示教育经历
- 展示工作经历
- 展示项目经历
- 支持 PDF 简历下载或浏览
- 页面需适配打印场景

### 5.5 博客功能
系统应支持博客内容发布与展示。

功能要求：
- 展示博客列表
- 展示博客详情
- 支持封面、摘要、正文、分类、标签
- 支持阅读量统计
- 支持后台新增、编辑、删除博客

### 5.6 动态功能
系统应支持动态内容发布与展示。

功能要求：
- 展示动态列表
- 支持动态详情查看
- 支持图文内容展示
- 支持后台新增、编辑、删除动态

### 5.7 作品管理与展示
系统应支持作品的维护与公开展示。

功能要求：
- 支持展示作品名称、简介、封面、多图、视频或外链
- 支持后台新增、编辑、删除作品
- 支持按排序与发布状态控制前台展示

### 5.8 荣誉管理与展示
系统应支持荣誉内容的维护与展示。

功能要求：
- 支持荣誉名称、授予机构、说明、封面、图集、证书附件维护
- 前台按时间线方式展示荣誉
- 支持附件或证书 PDF 打开查看
- 支持后台新增、编辑、删除荣誉

### 5.9 留言功能
系统应支持游客留言与后台管理。

功能要求：
- 游客可提交留言内容
- 系统记录留言来源信息
- 后台可查看全部留言
- 后台可编辑或删除留言
- 后台可维护留言页介绍文案

### 5.10 访问统计与日志
系统应具备基础运营统计能力。

功能要求：
- 记录公开页面访问量
- 统计全站访问次数
- 统计文章总浏览量
- 提供访问趋势展示
- 提供设备分布展示
- 提供热门页面排行
- 提供访问日志分页查看、筛选、删除能力

### 5.11 个人资料与简历后台维护
系统应支持统一维护个人资料。

功能要求：
- 维护站点标题
- 维护首页展示信息
- 维护关于我正文
- 维护教育经历和工作经历
- 维护联系方式
- 维护头像
- 维护简历 PDF 地址

### 5.12 首页 Hero 管理
系统应允许管理员独立配置首页首屏。

功能要求：
- 支持主标题、副标题、标签等内容配置
- 支持背景图和人物图配置
- 支持区块化 Hero 配置结构

### 5.13 足迹管理
系统应支持首页足迹展示与后台维护。

功能要求：
- 支持省/市/区县层级数据维护
- 支持访问时间、说明、排序、重要程度维护
- 前台按已发布状态展示足迹

### 5.14 媒体资源管理
系统应提供统一的媒体资源管理能力。

功能要求：
- 支持文件上传
- 支持历史文件导入
- 支持图片 / PDF 预览
- 支持获取媒体列表
- 支持删除文件
- 删除前必须校验引用关系，避免误删正在使用的资源

### 5.15 用户管理
系统应支持后台用户管理。

功能要求：
- 支持新增用户
- 支持编辑用户
- 支持删除用户
- 用户名必须唯一
- 至少保留一个管理员账号
- 当前登录管理员账号应有必要保护逻辑

### 5.16 系统配置
系统应支持基础运行参数配置。

功能要求：
- 支持本地存储和 OSS 存储切换
- 支持 OSS 相关参数配置
- 支持足迹区文案配置
- 支持页脚版权和技术支持文案配置

## 6. 非功能需求

### 6.1 安全性
- 后台接口必须基于 JWT 鉴权
- 不同角色应有不同权限范围
- 敏感配置应支持加密或通过环境变量传入
- 系统应具备基础 XSS 防护能力
- 系统应设置常见安全响应头
- 后台入口应进行一定程度的隐藏处理

### 6.2 易用性
- 前台页面需具备良好的可读性和视觉一致性
- 后台操作应直观，减少维护成本
- 关键操作应提供明确反馈

### 6.3 可维护性
- 前后端接口命名统一
- 文档应随实现同步更新
- 媒体、资料、配置应集中管理
- 初始化 SQL 和部署文档应明确

### 6.4 性能
- 首页聚合接口应尽量减少前端多次请求成本
- 高频公开接口应具备基础性能优化能力
- 媒体资源访问应稳定可靠

### 6.5 兼容性
- 前台需兼容桌面端和移动端
- 简历页面需兼容打印场景
- 上传与资源访问需兼容本地与 OSS 两种存储方式

## 11. 新增规划：多租户博客平台能力

为后续将 `dyx-blog` 升级为支持多人注册使用的多租户个人博客平台，补充如下规划（暂不实施，仅作为后续版本演进方向）：

### 11.1 多站点与路径路由
- 引入“站点 / 个人博客”抽象，支持一个平台托管多个独立站点
- 每个站点通过路径访问形式暴露，例如 `/u/{siteSlug}` 作为站点前台入口
- 站点下的核心前台页面示例：
  - `/u/{siteSlug}`：站点首页
  - `/u/{siteSlug}/about`：关于我
  - `/u/{siteSlug}/resume`：简历
  - `/u/{siteSlug}/blog`：博客列表
  - `/u/{siteSlug}/blog/:id`：博客详情
  - `/u/{siteSlug}/moments`：动态列表
  - `/u/{siteSlug}/guestbook`：留言页
- 保留当前根路径作为平台默认站点或主入口

### 11.2 功能全开放的站点能力
- 新增“站点”级别的数据模型，每个站点拥有独立的：
  - 个人资料（站点标题、首页 Hero、关于我、联系方式等）
  - 博客、动态、作品、荣誉、足迹
  - 媒体资源与访问统计
- 在现有 `dyx_profile`、`dyx_post`、`dyx_moment`、`dyx_project`、`dyx_work`、`dyx_honor`、`dyx_guestbook_message`、`dyx_footprint`、`dyx_media`、`dyx_site_visit_stat`、`dyx_site_visit_log`、`dyx_system_config` 等表基础上，后续版本规划通过增加 `site_id` 字段实现按站点隔离
- 站点功能默认全开放，与当前个人站点功能范围保持一致

### 11.3 注册与站点初始化
- 新增平台级注册入口，支持用户通过账号密码方式注册，无需邮箱校验或人工审核
- 注册成功后自动创建一个默认站点，并为其生成唯一 `siteSlug`（可基于用户名派生）
- 引导新用户进入站点配置向导，完成基础资料（站点标题、关于我、联系方式、主题配置等）初始化
- 后续可按需扩展邮箱校验、邀请注册等机制（当前仅作为预留方向）

## 12. 验收标准（概要）
项目可视为满足当前阶段需求，至少需要达到：
- 游客可完整访问并浏览公开站点核心页面
- 管理员可成功登录后台并完成内容维护
- 媒体上传、引用、删除逻辑完整可用
- 站点访问量和博客阅读量可正常统计
- 简历 PDF 可正常下载或预览
- 前后端主要页面在桌面端与移动端均可使用
- 部署后可通过 Nginx + Spring Boot + MySQL 正常运行
