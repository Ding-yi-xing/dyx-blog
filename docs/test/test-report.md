# dyx-blog 测试报告

## 1. 测试概述
本次测试依据 `docs/requirements.md`、`docs/api-design.md`、`docs/project-usage.md`、`docs/database-design.md` 对 dyx-blog 项目进行了系统性测试评估。测试目标覆盖功能、性能、兼容性、安全性与用户体验五个维度。

本次测试实际采用了以下方式：
- 需求文档分析
- 前端源码静态测试
- 前端构建验证
- 服务器环境公开页面与公开 API 冒烟
- 服务器环境后台登录、鉴权、管理接口读写验证
- 服务器环境补充边界测试
- 数据库只读抽样核验
- 缺陷与风险归档

### 3.2 前台测试结果
#### 通过项
- 公开页面 `/`、`/about`、`/resume`、`/blog`、`/moments`、`/guestbook` 可访问，返回 200
- 公开接口 `/api/site/home`、`/profile`、`/posts`、`/moments`、`/projects`、`/works`、`/honors`、`/footprints`、`/guestbook` 均返回 200
- 博客详情接口可正常返回，且阅读量会递增
- 动态详情接口可正常返回真实存在的数据
- 留言提交成功；私有留言不会出现在公开留言列表
- 留言边界校验生效：空内容、空格、缺少 content、超长内容均返回 400
- 登录接口对缺失密码、空字符串和明显恶意样式输入均返回 400
- 页面访问统计接口 `POST /api/site/visit/home` 返回 200
- OSS 外部图片资源大多可正常访问

#### 失败项
- 简历 PDF 资源访问失败，`resumePdfUrl` 对应 `/media/*.pdf` 返回 500
- 返回数据存在系统性中文乱码
- 作品/项目部分外链缺少协议头，存在错误跳转风险
- 作品 `videoUrl` 字段实际指向图片资源
- 不存在的博客详情返回 `200 + data:null`，不存在的动态详情返回 `400`，错误语义不一致
- 公开文章列表对 `page=-1`、`pageSize=0`、`pageSize=1000` 等异常分页参数均返回 200，分页边界约束不足
- 留言接口在错误请求方法与错误字段类型输入下会触发 500
- HTML 页面响应头未见与 API 等同级别的完整安全头覆盖

- 后台用户列表对 `pageSize=0/1000` 仍返回完整数据，分页上限控制不足
- 后台访问日志对无效筛选值可安全返回空列表

### 3.4 数据库抽样核验结果
#### 通过项
- MySQL 可成功连接，库为 `dyx_blog`，版本 `8.4.8`
- 数据库、表、文本列字符集与排序规则均为 `utf8mb4 / utf8mb4_0900_ai_ci`
- 数据库抽样与接口返回能够相互印证，说明接口层并未单独篡改这些问题数据

#### 发现项
- 业务表实际命名为 `dyx_user`、`dyx_post`、`dyx_moment`、`dyx_work`、`dyx_project`、`dyx_profile`、`dyx_guestbook_message` 等
- `dyx_post`、`dyx_moment`、`dyx_project`、`dyx_profile`、`dyx_work` 抽样记录中已存在明显乱码，说明脏数据已写入数据库
- `dyx_work.video_url` 抽样值对应 `.jpg` 图片，和接口异常表现一致
- `dyx_work.work_link`、`dyx_project.project_link` 中存在 `www.baidu.com` 这类无协议链接，和接口表现一致
- `dyx_profile.resume_pdf_url` 指向媒体路径，但线上访问仍返回 500，说明问题更可能在媒体映射/文件本体而非数据库字段缺失

## 6. 风险评估
### 6.1 高风险
- 开放重定向问题若不修复，不建议上线后台登录链路
- 简历 PDF 不可访问，直接影响核心需求验收
- 用户管理与文章管理保存异常，影响后台核心运营功能
- 数据库中已存在乱码脏数据，若不清洗会持续污染前后台展示与后续运营内容

### 6.2 中风险
- 外链缺少协议头可能导致跳转错误
- 首页地图与复杂交互可能在低性能设备上出现体验问题
- 构建产物偏大，可能影响移动端与弱网体验
- 接口对异常分页参数和错误请求处理不稳健，可能增加误用与异常流量风险

### 6.3 低风险
- PDF 与自动化依赖缺失主要影响测试效率，不直接等同于线上生产缺陷

## 7. 质量结论
### 7.1 当前结论
系统已具备较完整的公开站点与后台管理能力，且绝大多数查询接口、登录鉴权、留言、访问统计、动态/荣誉/作品/项目/足迹/配置等链路可正常工作；但同时存在多个关键缺陷，尤其是后台保存链路异常、简历 PDF 故障、数据库层乱码脏数据和安全问题，这些缺陷足以影响上线质量判断。

### 7.2 上线建议
当前结论：**不通过**

原因：
- 存在 P1 安全问题（开放重定向）
- 存在多个 P1 功能问题（简历 PDF、用户保存、文章保存、错误语义异常）
- 存在数据库层已落库的系统性乱码问题，影响前后台核心展示与数据质量

## 8. 改进建议
### 8.1 必须优先处理
- 修复后台登录 redirect 校验问题，仅允许站内安全跳转
- 修复 `resumePdfUrl` 对应的媒体资源访问失败问题，重点排查文件名编码、静态资源映射和文件存在性
- 修复后台用户保存接口与文章保存接口 500
- 修复文章更新不存在资源仍返回成功的问题
- 修复留言接口与媒体预览接口对异常输入触发 500 的问题
- 全面排查数据录入/导入链路编码，并清洗数据库中的历史乱码脏数据

### 8.2 建议尽快处理
- 为作品/项目外链统一补全协议并在后台录入时校验
- 校验 `videoUrl` 字段真实资源类型，避免图片误填为视频
- 统一详情接口不存在资源时的错误语义
- 为 HTML 页面补充与 API 同等级别的安全头策略
- 对地图、图表、后台管理模块做进一步拆包与懒加载
- 优化首页滚动交互，减少强制滚轮劫持
- 为分页参数增加最小值、最大值校验

### 8.3 中长期建议
- 建立单元测试 + 集成测试 + E2E 测试体系
- 增加数据库级自动化校验和回归机制
- 增加性能基线、Lighthouse 与接口压测
- 增加安全测试清单，包括认证绕过、资源越权、上传安全、XSS/CSRF 验证
