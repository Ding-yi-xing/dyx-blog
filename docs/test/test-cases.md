## 4. 安全测试用例

### ST-001 登录跳转安全性
- 测试步骤：构造 `/dyx-manager/login?redirect=https://evil.example`
- 预期结果：仅允许站内安全路径跳转
- 实际结果：源码直接信任 query 参数并 `router.push`，存在开放重定向风险
- 结论：不通过

### ST-002 后台接口鉴权
- 测试步骤：未登录访问后台接口
- 预期结果：返回 401
- 实际结果：后台接口未登录返回 401
- 结论：通过

### ST-003 Token 存储安全性
- 测试步骤：检查登录态存储方式
- 预期结果：有明确风控措施
- 实际结果：JWT 存储于 `localStorage`，若出现 XSS 会放大影响面
- 结论：风险项

### ST-004 响应头安全性
- 测试步骤：检查 API 与 HTML 页面响应头
- 预期结果：前后端页面与接口均有较完整安全头
- 实际结果：API 安全头较完整，HTML 页面未见同等级别完整覆盖
- 结论：部分通过
