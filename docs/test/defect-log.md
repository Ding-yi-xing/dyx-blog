# dyx-blog 缺陷清单

## 缺陷统计
- P1：4 个
- P2：5 个
- P3：2 个
- 风险项：2 个

## DEF-001 登录成功后存在开放重定向漏洞
- 等级：P1 / 安全
- 模块：后台登录
- 位置：`frontend/src/views/admin/AdminLoginView.vue`
- 现象描述：登录成功后直接使用 `route.query.redirect` 作为跳转目标，未校验是否为站内路径。
- 复现步骤：
  1. 访问 `/dyx-manager/login?redirect=https://evil.example`
  2. 输入合法账号密码并登录
  3. 观察登录后的跳转行为
- 预期结果：仅允许跳转到站内受信路径，如 `/dyx-manager/**`
- 实际结果：存在跳转到外部地址的风险
- 影响分析：可被用于钓鱼链路拼接，影响账号安全感知与系统可信度
- 建议：限制 redirect 仅允许以 `/` 开头且命中白名单路由

## DEF-002 简历 PDF 资源访问失败
- 等级：P1 / 功能
- 模块：简历
- 现象描述：`GET /api/site/profile` 返回的 `resumePdfUrl` 实际访问返回 500，无法下载或预览。
- 复现步骤：
  1. 调用 `GET /api/site/profile`
  2. 获取 `resumePdfUrl`
  3. 请求对应 `/media/*.pdf` 资源
- 预期结果：返回 PDF 文件流，支持在线预览或下载
- 实际结果：接口返回 500，无法获取简历 PDF
- 影响分析：直接阻断简历 PDF 下载/浏览需求
- 建议：排查媒体映射、文件存在性、文件名编码及静态资源读取逻辑

## DEF-003 后台用户保存接口 500
- 等级：P1 / 功能
- 模块：用户管理
- 现象描述：后台用户编辑接口在使用原数据回写时仍返回 500。
- 复现步骤：
  1. 使用管理员账号登录
  2. 调用 `GET /api/dyx-manager/users`
  3. 取已有用户原始数据，调用 `PUT /api/dyx-manager/users/{id}`
- 预期结果：返回 200，保存成功
- 实际结果：返回 500
- 影响分析：用户编辑能力不可用，影响后台核心管理功能
- 建议：检查用户保存 DTO、密码处理、空字段处理和数据库更新逻辑

## DEF-004 后台文章保存接口 500
- 等级：P1 / 功能
- 模块：博客管理
- 现象描述：后台文章编辑接口在使用原数据回写时返回 500。
- 复现步骤：
  1. 使用管理员账号登录
  2. 调用 `GET /api/dyx-manager/posts`
  3. 取已有文章原始数据，调用 `PUT /api/dyx-manager/posts/{id}`
- 预期结果：返回 200，保存成功
- 实际结果：返回 500
- 影响分析：文章编辑能力不可用，影响核心内容运营
- 建议：检查文章更新接口字段映射、校验和数据库更新逻辑

## DEF-005 前端关键页面打包产物过大
- 等级：P2 / 性能
- 模块：首页、主包、后台仪表盘
- 现象描述：Vite 构建告警显示多个 chunk 超过 500 kB。
- 证据：
  - `HomeView` 约 1464.72 kB
  - `index` 约 1033.65 kB
  - `AdminDashboardView` 约 545.04 kB
- 预期结果：关键 chunk 体积受控，首屏加载与弱网体验可接受
- 实际结果：存在明显大包风险
- 影响分析：影响首屏时间、弱网访问、移动端体验和缓存效率
- 建议：进一步拆分地图/图表/管理模块，按需加载重型依赖

## DEF-006 首页滚轮强制分屏交互存在可用性问题
- 等级：P2 / 用户体验
- 模块：首页
- 位置：`frontend/src/views/web/HomeView.vue`
- 现象描述：首页拦截 `wheel` 事件并强制切屏，可能导致触控板、辅助设备或精细滚动场景下体验不佳。
- 预期结果：滚动自然、可中断、兼顾无障碍
- 实际结果：滚动被劫持，可能产生跳屏与操作不连贯
- 影响分析：影响桌面端高精度触控板用户与可访问性
- 建议：弱化强制滚动，增加用户可控性并验证键盘/屏幕阅读器路径

## DEF-007 服务器返回数据存在系统性中文乱码
- 等级：P2 / 数据质量
- 模块：前台公开接口、后台管理接口
- 现象描述：文章、动态、足迹、联系方式、简历文件名等多个中文字段返回乱码。
- 预期结果：中文内容正确显示
- 实际结果：多处中文字段乱码
- 影响分析：影响内容展示、链接可读性、文件访问与整体专业度
- 建议：排查数据库字符集、连接字符集、JDBC 参数、服务端响应编码及历史脏数据

## DEF-008 作品/项目外链缺少协议头
- 等级：P2 / 功能
- 模块：作品展示、项目展示
- 现象描述：接口中 `workLink`、`projectLink` 存在 `www.baidu.com` 这类无协议 URL。
- 预期结果：外链应为合法绝对 URL，如 `https://...`
- 实际结果：存在未带协议的链接
- 影响分析：前端可能拼接当前站点地址，导致跳转错误
- 建议：后端或前端统一补全协议，并在后台录入时增加校验

## DEF-009 作品 videoUrl 字段实际为图片资源
- 等级：P2 / 数据质量
- 模块：作品展示
- 现象描述：`videoUrl` 指向 `.jpg`，且返回 `Content-Type: image/jpeg`。
- 预期结果：videoUrl 应为视频资源
- 实际结果：字段语义与实际资源不匹配
- 影响分析：前端若按视频渲染将出现异常或体验问题
- 建议：清洗存量数据并在后台录入时校验 MIME/扩展名

## DEF-010 详情接口不存在资源时返回语义不一致
- 等级：P2 / 接口一致性
- 模块：博客详情、动态详情
- 现象描述：不存在的博客详情返回 `200 + data:null`，不存在的动态详情返回 `400`。
- 预期结果：同类详情接口对不存在资源应采用一致的错误语义
- 实际结果：错误码与返回结构不一致
- 影响分析：增加前端分支处理复杂度，降低接口一致性
- 建议：统一不存在/未发布资源的返回策略

## DEF-011 PDF 需求文档读取链路依赖缺失
- 等级：P3 / 环境
- 模块：测试环境
- 现象描述：PDF 读取依赖 `pdftoppm` 不可用，无法直接读取 PDF。
- 预期结果：可直接读取 PDF 进行评审
- 实际结果：改为读取同名 Markdown 源文档完成需求分析
- 影响分析：不影响文档内容分析结论，但说明环境不完整
- 建议：补齐 Poppler 相关依赖

## DEF-012 浏览器自动化环境缺失
- 等级：P3 / 环境
- 模块：自动化测试
- 现象描述：Python `playwright` 模块缺失，无法执行浏览器自动化脚本。
- 预期结果：可执行 Playwright 冒烟测试
- 实际结果：仅能完成静态分析与命令级验证
- 影响分析：降低回归效率与可重复性
- 建议：补充 Playwright 运行时与浏览器依赖

## DEF-013 公共分页参数缺少边界约束
- 等级：P2 / 接口鲁棒性
- 模块：公开文章列表
- 现象描述：`/api/site/posts` 对 `page=-1`、`pageSize=0`、`pageSize=1000` 等异常分页参数均返回 200，且仍返回完整列表。
- 复现步骤：
  1. 调用 `GET /api/site/posts?page=-1&pageSize=10`
  2. 调用 `GET /api/site/posts?page=1&pageSize=0`
  3. 调用 `GET /api/site/posts?page=1&pageSize=1000`
- 预期结果：应限制分页参数取值或返回 400
- 实际结果：异常参数被静默接受
- 影响分析：增加接口误用风险，也可能放大批量数据返回带来的性能压力
- 建议：对 `page/pageSize` 做最小值和最大值校验

## DEF-014 留言提交接口异常输入可触发 500
- 等级：P1 / 功能鲁棒性
- 模块：留言
- 现象描述：`GET /api/site/guestbook/messages` 与错误类型 JSON 提交均可触发 500，而非稳定的 400 级错误。
- 复现步骤：
  1. 以 `GET` 方式访问 `POST /api/site/guestbook/messages`
  2. 提交 `{"nickname":123,"content":["x"],"isPrivate":"no"}`
- 预期结果：返回 400/405 等明确客户端错误
- 实际结果：返回 500
- 影响分析：说明接口参数校验与方法约束不健壮，可能放大异常流量影响
- 建议：补充请求方法限制、DTO 类型校验与统一异常映射

## DEF-015 后台文章更新不存在 ID 仍返回成功
- 等级：P1 / 功能
- 模块：博客管理
- 现象描述：`PUT /api/dyx-manager/posts/999999999` 对不存在文章 ID 返回 200，并回显该 ID 的对象。
- 复现步骤：
  1. 使用管理员账号登录
  2. 调用 `PUT /api/dyx-manager/posts/999999999`
  3. 提交最小文章对象 `{id,title,summary,content}`
- 预期结果：不存在资源应返回 404 或明确失败
- 实际结果：返回 200，语义表现为保存成功
- 影响分析：破坏后台数据修改语义，前端无法可靠区分真实更新与异常写入
- 建议：更新前先校验资源存在性，并统一不存在资源返回策略

## DEF-016 媒体预览缺失必填参数返回 500
- 等级：P2 / 功能鲁棒性
- 模块：媒体管理
- 现象描述：访问 `/api/dyx-manager/media/content` 且不传 `fileUrl` 时返回 500。
- 复现步骤：
  1. 使用管理员账号登录
  2. 调用 `GET /api/dyx-manager/media/content`
- 预期结果：返回 400，提示缺失必要参数
- 实际结果：返回 500
- 影响分析：参数缺失场景被错误升级为服务端异常，影响接口可观测性与稳定性
- 建议：补充必填参数校验并统一转换为 400

## DEF-017 数据库内已存在乱码脏数据
- 等级：P1 / 数据质量
- 模块：数据库内容层
- 现象描述：数据库 `dyx_blog` 库、表、文本列字符集均为 `utf8mb4`，但 `dyx_post`、`dyx_moment`、`dyx_project`、`dyx_profile`、`dyx_work` 抽样记录中已存在大量乱码文本。
- 复现步骤：
  1. 连接 `118.178.186.196:3306/dyx_blog`
  2. 执行只读查询抽样 `dyx_post`、`dyx_moment`、`dyx_project`、`dyx_profile`、`dyx_work`
  3. 对比字段值与接口返回
- 预期结果：数据库原始中文内容可正常读取
- 实际结果：数据库记录本身已是乱码，且与接口返回一致
- 影响分析：说明问题不只是 HTTP 响应编码，历史脏数据已写入数据库，前后台都会持续受影响
- 建议：排查导入/录入链路编码，清洗现有脏数据，并对写入链路增加编码回归校验
