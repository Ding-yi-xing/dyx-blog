# dyx-blog 测试计划

## 1. 测试目标
基于 `docs/requirements.md`、`docs/api-design.md`、`docs/project-usage.md`、`docs/database-design.md` 对 dyx-blog 项目开展系统测试，验证其功能正确性、稳定性、安全性、兼容性、性能表现与用户体验质量。

## 2. 测试依据
- 需求文档：`docs/requirements.md`
- 接口设计：`docs/api-design.md`
- 使用说明：`docs/project-usage.md`
- 数据库设计：`docs/database-design.md`
- 项目进度：`docs/project-progress.md`
- 前端源码：`frontend/src/**`

## 3. 测试范围
### 3.1 功能测试
- 前台：首页、关于我、简历、留言、动态、博客
- 后台：登录、仪表盘、访问日志、博客管理、动态管理、荣誉管理、首页 Hero、首页足迹、关于我、简历、作品、媒体、系统配置、用户管理
- 公共能力：JWT 登录态、路由守卫、访问统计、媒体预览、PDF 简历下载/打印

### 3.2 非功能测试
- 性能：前端构建产物规模、关键页面资源体积、潜在高负载点
- 兼容性：桌面端/移动端/打印场景、前后台路由兼容性
- 安全性：认证授权、XSS、开放重定向、Token 存储、资源访问与配置暴露
- 用户体验：导航可达性、空状态、错误提示、表单反馈、交互一致性

## 4. 测试策略
### 4.1 测试方法
- 需求评审
- 静态测试：文档与源码一致性分析
- 构建验证：`frontend npm run build`
- 冒烟测试：关键入口、关键模块、关键交互逻辑检查
- 风险测试：围绕登录跳转、媒体引用、地图渲染、大体积页面进行专项分析

### 4.2 测试设计方法
- 等价类划分
- 边界值分析
- 状态迁移
- 错误猜测
- 场景法
- 基于风险的测试

## 5. 测试环境
### 5.1 实际可用环境
- OS：Windows 11 Pro 10.0.26200
- 前端：Vue 3 + TypeScript + Vite
- Node 依赖：已安装
- 前端构建：成功

### 5.2 环境限制
- 当前仓库仅存在 `frontend/` 与 `docs/`，`backend/` 目录缺失
- 后端服务、数据库、Nginx 代理环境未提供，无法执行真实接口联调、性能压测、鉴权联调、数据库验证
- PDF 直接读取依赖缺失（`pdftoppm` 不可用），本次需求分析改为读取同目录 Markdown 源文档
- Python Playwright 未安装，无法执行浏览器自动化脚本
- `vite preview` 启动失败，提示 `dist` 不存在；该现象与此前构建成功结果不一致，需进一步核实本地产物状态

## 6. 进入/退出准则
### 6.1 进入准则
- 需求文档可读取
- 源码可读取
- 前端依赖可构建

### 6.2 退出准则
- 核心模块完成静态/构建/逻辑验证
- 缺陷已记录并分级
- 风险与改进建议已输出
- 形成正式测试报告

## 7. 缺陷分级标准
- P0：系统不可用/高危安全问题/核心链路完全阻断
- P1：核心功能异常或严重安全风险，影响上线
- P2：重要功能缺陷，有替代路径或局部影响
- P3：一般问题，主要影响体验、文案、非关键交互或可维护性
