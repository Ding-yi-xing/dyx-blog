# dyx-blog 项目使用说明

## 1. 文档目的
本文档面向项目维护者、部署人员和后台运营人员，说明 `dyx-blog` 项目的本地运行、生产部署、后台使用与常见维护方式。

## 2. 项目简介
`dyx-blog` 是一个前后台一体化个人站点项目，覆盖：
- 个人主页展示
- 关于我 / 简历展示
- 博客与动态发布
- 留言互动
- 后台内容管理
- 媒体资源管理
- 访问统计与访问日志查看

技术栈：
- 前端：Vue 3、TypeScript、Pinia、Vue Router、Axios、Tailwind CSS、Element Plus
- 后端：Spring Boot 3、MyBatis-Plus、MySQL、JWT
- 文件存储：本地存储 / 阿里云 OSS

## 3. 目录结构
```text
backend/    Spring Boot 后端
frontend/   Vue 3 前端
docs/       项目文档
```

## 4. 本地开发使用

### 4.1 环境要求
- JDK 17
- Maven 3.9+
- Node.js 18+
- npm 9+
- MySQL 8.0+

### 4.2 初始化数据库
1. 创建数据库：`dyx_blog`
2. 导入初始化脚本：
   - `backend/src/main/resources/sql/dyx-blog-init.sql`

说明：项目当前已有完整初始化 SQL，旧的 `schema.sql` 已不再作为当前主入口使用。

### 4.3 启动后端
在 `backend` 目录执行：

```bash
mvn spring-boot:run
```

或打包后运行：

```bash
mvn clean package
java -jar target/dyx-blog-1.0.0.jar
```

默认后端端口：`8080`

### 4.4 启动前端
在 `frontend` 目录执行：

```bash
npm install
npm run dev
```

默认开发地址通常为：
- `http://localhost:5173`

### 4.5 本地访问入口
前台页面：
- 首页：`/`
- 关于我：`/about`
- 简历：`/resume`
- 留言：`/guestbook`
- 动态：`/moments`
- 博客：`/blog`

后台页面：
- 登录页：`/dyx-manager/login`
- 管理后台：`/dyx-manager`

说明：
- 公开站点导航中不再直接暴露后台入口。
- 当前实现保留隐藏入口：在首页点击站点标题 5 次后跳转到后台登录页。

## 5. 生产部署使用

### 5.1 后端部署
生产部署建议参考：`docs/DEPLOY.md`

关键点：
- 使用 Java 17
- 使用 MySQL 8+
- 使用环境变量传入数据库、JWT 和加密密钥
- 后端上传目录使用 `uploads/`
- 不要在生产环境直接暴露 8080 端口

推荐通过环境变量启动：

```bash
export MYSQL_HOST=localhost
export MYSQL_PORT=3306
export MYSQL_USER=your_user
export MYSQL_PWD=your_password
export JWT_SECRET=replace_with_secure_secret
export ENCRYPT_KEY=replace_with_32_char_key

nohup java -jar dyx-blog-1.0.0.jar --spring.profiles.active=prod > logs/stdout.log 2>&1 &
```

### 5.2 前端部署
在 `frontend` 目录执行：

```bash
npm install
npm run build
```

将生成后的 `dist/` 部署到 Nginx 静态目录。

### 5.3 Nginx 代理要点
- 前端静态资源由 Nginx 托管
- `/api/` 反向代理到 Spring Boot
- 刷新路由时需回退到 `index.html`
- 生产环境必须正确透传 `X-Forwarded-For` 或 `X-Real-IP`

## 6. 前台使用说明

### 6.1 首页
首页展示：
- Hero 首屏信息
- 最新博客
- 最新动态
- 精选项目
- 最新荣誉
- 足迹地图

维护方式：
- Hero 信息由后台“首页 Hero”配置
- 足迹区文案由“系统配置”维护
- 足迹点位由“首页足迹”维护

### 6.2 关于我
展示内容：
- 头像
- 个人介绍
- 联系方式
- 作品列表
- 荣誉时间线

维护方式：
- 基础资料在“关于我管理”维护
- 作品在“作品管理”维护
- 荣誉在“荣誉管理”维护

### 6.3 简历
展示内容：
- 基础资料
- 教育经历
- 工作经历
- 项目经历
- PDF 简历下载/查看

维护方式：
- 教育/工作经历在“关于我管理/简历管理”维护
- 项目经历在“简历管理”对应项目模块维护
- PDF 简历通过媒体库上传后绑定到资料字段

### 6.4 留言页
访客可提交留言。
后台可：
- 修改留言页介绍文案
- 查看全部留言
- 控制留言内容
- 删除留言

### 6.5 博客与动态
- 博客用于长文内容展示
- 动态用于时间流式内容展示
- 博客详情会累加阅读量
- 公开页面会记录访问量

## 7. 后台使用说明

### 7.1 登录
后台登录页为 `/dyx-manager/login`。
登录成功后进入仪表盘。

说明：
- 只有具备后台角色的账号才允许登录后台。
- 公开站点不再展示显式后台入口，日常维护请直接访问登录路由或使用隐藏入口。

### 7.2 仪表盘
可查看：
- 文章数量
- 动态数量
- 荣誉数量
- 用户数量
- 博客总浏览量
- 全站访问量
- 访问趋势
- 设备分布
- 热门页面

### 7.3 访问日志
可按以下条件筛选：
- 时间范围
- 页面标识
- 设备类型
- IP 地址

支持：
- 分页查看
- 单条删除
- 批量删除

时间说明：
- 后端访问日志时间统一按 `Asia/Shanghai`（北京时间）写入与返回，避免部署在 UTC 等服务器时出现慢 8 小时的问题

权限说明：
- 访问日志查看可进入后台页面后使用。
- 单条删除与批量删除仅 `ADMIN` 角色可执行。

### 7.4 内容管理
后台主要内容管理模块：
- 博客管理
- 动态管理
- 荣誉管理
- 作品管理
- 留言管理
- 用户管理

常规操作：
- 新建
- 编辑
- 删除
- 发布 / 草稿控制（视数据字段而定）

### 7.5 首页配置
包含：
- 首页 Hero 配置
- 首页足迹配置
- 首页活动区（若页面启用）

### 7.6 关于我 / 简历管理
用于维护：
- 站点标题
- 头像
- 联系方式
- 关于我正文
- 教育经历
- 工作经历
- PDF 简历

图片与文件选择说明：
- 媒体库负责上传、选择与预览原始资源。
- 头像、首页首屏背景图、首页右侧人物图等业务图片支持“直接使用”或“裁剪后使用”。
- 裁剪能力在具体业务表单中完成，媒体库本身不强制先裁剪再上传。

### 7.7 媒体资源管理
支持：
- 上传文件
- 导入 `backend/uploads` 已存在文件
- 图片 / PDF 预览
- 删除文件

注意：
- 删除前系统会检查资源引用关系
- 正在被文章、荣誉、作品、资料等使用的资源不允许直接删除
- 媒体库只负责原始资源管理；业务裁剪在头像、首页首屏等具体页面完成

### 7.8 系统配置
可配置：
- 存储方式（本地 / OSS）
- OSS 参数
- 足迹区文案
- 页脚版权与技术支持文案

## 8. 权限与安全使用说明
- 后台接口通过 JWT 鉴权
- 只有 `ADMIN` 角色可登录后台
- 系统配置、用户管理、访问日志删除等能力仅 `ADMIN` 可执行
- 前端登录态默认保存在 `sessionStorage`，关闭当前浏览器会话后需重新登录
- 后台用户、荣誉、作品等 Long 主键在前后端交互中按字符串安全传递，避免 JavaScript 精度丢失
- 建议仅开放 80/443 给公网
- 后端 8080 只允许反向代理访问
- 数据库、JWT 密钥、加密密钥必须使用环境变量管理
- 生产环境请启用 HTTPS

## 9. 常见维护场景

### 9.1 更新首页展示内容
1. 登录后台
2. 进入“首页 Hero”修改文案和图片
3. 进入“首页足迹”维护足迹数据
4. 进入“系统配置”调整足迹区文案和页脚文案

### 9.2 发布新博客
1. 进入“博客管理”
2. 新建文章并填写标题、摘要、正文、封面等
3. 设为已发布
4. 前台博客列表和详情页即可展示

### 9.3 更新简历 PDF
1. 进入“媒体资源管理”上传 PDF
2. 进入“简历管理”或资料管理页面
3. 将 `resumePdfUrl` 绑定到新的 PDF 文件地址
4. 前台简历页可下载/查看最新版本

### 9.4 导入历史上传文件
1. 确保文件已存在于 `backend/uploads`
2. 在后台媒体管理执行“导入已有文件”
3. 系统会扫描并导入到媒体库记录中

## 10. 常见问题

### 10.1 刷新页面 404
原因：Nginx 未做 SPA 路由回退。
处理：增加 `try_files $uri $uri/ /index.html;`

### 10.2 访问日志 IP 不准确
原因：代理未透传真实 IP 头。
处理：检查 `X-Forwarded-For` / `X-Real-IP` 配置。

### 10.3 媒体删除失败
原因：文件仍被业务模块引用。
处理：先解除文章、荣誉、资料等引用后再删除。

### 10.4 前端请求 401
原因：Token 失效或未登录。
处理：重新登录后台。
