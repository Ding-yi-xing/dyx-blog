 Context

 服务器环境下博客详情和动态详情无法正常展示，优先要排查并修复最可能的线上问题，同时避免做过度改动。当前代码里最明显的问
 题是博客前台 API
 路径与后端公开接口不一致；另外详情页没有错误处理，线上接口失败时会退化成空内容，用户看起来像“没展示出来”。由于前端使用
  createWebHistory()，部署侧还必须保证深链接路由回退到 index.html。

 Recommended approach

 1. 先修正博客前台 API 路径，确保与后端公开接口一致。
   - 关键文件：frontend/src/api/modules/site.ts
   - 将博客列表/详情保持为 /site/posts、/site/posts/:id，与后端
 backend/src/main/java/com/dyx/blog/controller/publics/SiteController.java 中 /api/site/posts、/api/site/posts/{id}
 对齐。
   - 复用现有 http 封装：frontend/src/api/http.ts
 2. 为博客详情和动态详情增加最小错误态处理，避免线上接口失败时页面只显示默认占位文本。
   - 关键文件：
       - frontend/src/views/web/BlogDetailView.vue
     - frontend/src/views/web/MomentDetailView.vue
   - 增加 loading / errorMessage 状态，在请求失败、返回空数据或未发布时给出明确提示。
   - 继续复用现有详情拉取函数：
       - getPostDetail / getMomentDetail in frontend/src/api/modules/site.ts
   - 如有需要，可复用错误提取工具：frontend/src/utils/error.ts
 3. 核对动态详情接口行为，若前端错误态仍不足以覆盖，再调整后端返回语义。
   - 关键文件：backend/src/main/java/com/dyx/blog/service/impl/SiteServiceImpl.java
   - 现状：getMomentDetail 对未发布/不存在数据抛出业务异常；getPostDetail 会在读取详情时递增浏览量。
   - 第一轮先不改后端逻辑，只通过前端错误态消除“空白/假成功”体验；若实际联调发现博客详情因浏览量更新失败导致
 500，再将浏览量更新改成非阻塞失败不影响正文返回。
