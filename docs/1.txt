 Context

 用户希望首页足迹地图满足两个要求：
 1. 默认缩放下不显示其他国家名称标签，只有继续缩小到更远视角时才逐步显示。
 2. 当前每次缩小时都会重新绘制边界线和足迹高亮区域，导致交互明显卡顿；需要优化为缩放时尽量复用已有覆盖物，而不是重复销
 毁和重建。

 当前实现集中在 frontend/src/components/web/HomeFootprintMap.vue，地理数据缓存主要在
 frontend/src/utils/footprintGeo.ts。已有缓存包括
 normalizedProvinceCache、boundaryCache，组件内也已有一部分“初始化一次后 show/hide”的模式，可直接复用扩展。

 Recommended approach

 1. 调整国家名称标签显示阈值

 文件：frontend/src/components/web/HomeFootprintMap.vue
 - 将国家标签显示条件从当前默认缩放附近进一步下调，只在比默认更远的缩放级别才显示。
 - 保持默认进入地图时不出现其他国家名称；只有用户继续缩小地图时，renderCityNameLabels 中的国家标签分支才生效。
 - 复用现有 COUNTRY_LABEL_MAX_ZOOM / shouldShowCountryLabels() 机制，不新增额外状态。

 2. 消除缩放时的重复渲染触发

 文件：frontend/src/components/web/HomeFootprintMap.vue
 - 移除 adjustMapZoom() 中对 renderMapOverlays() 的直接调用。
 - 统一依赖现有 zoomend / moveend 事件来触发渲染，避免一次缩放同时触发“主动渲染 + 事件渲染”两遍。
 - 保留 dragstart / zoomstart 中对 hasUserMoved 的处理。

 3. 将多边形覆盖物改为“初始化一次，后续显隐/按需重建”

 文件：frontend/src/components/web/HomeFootprintMap.vue
 - 重点优化：
   - renderProvinceOutlineOverlays
   - renderWorldCountryBoundaryOverlays
   - renderVisitedCityOverlays
   - renderCityBoundaryOverlays
 - 策略：
   - 对仅随主题变化或数据变化才需要变动的覆盖物，首次创建后缓存实例；缩放时不再 clear + new。
   - 对只受缩放阈值影响的覆盖物，改为 show()/hide() 控制，而不是每次 remove/add。
   - 仅当 props.items、visitedProvinceNames 或 theme 变化时，才销毁并重建对应覆盖物。

 4. 区分“数据变更渲染”和“缩放渲染”

 文件：frontend/src/components/web/HomeFootprintMap.vue
 - 将当前统一的 renderMapOverlays() 拆成更明确的调度：
   - 一类处理静态/半静态覆盖物（世界边界、省界、足迹高亮区域）
   - 一类处理随缩放变化更频繁的内容（标签、marker 可见性）
 - 缩放时只更新必要层；数据变化时才全量同步。
 - 这样可以避免每次缩放都重建省界和足迹高亮区域。

 5. 优化标签与 marker 的更新方式

 文件：frontend/src/components/web/HomeFootprintMap.vue
 - renderCityNameLabels()：避免每次缩放都 clearLabelOverlays() 后重新 new 全部 AMap.Text；优先改为按阈值

     Context

     用户希望首页足迹地图满足两个要求：
     1. 默认缩放下不显示其他国家名称标签，只有继续缩小到更远视角时才逐步显示。
     2. 当前每次缩小时都会重新绘制边界线和足迹高亮区域，导致交互明显卡顿；需要优化为缩放时尽量复用已有覆盖物，而不是重复
     销毁和重建。

     当前实现集中在 frontend/src/components/web/HomeFootprintMap.vue，地理数据缓存主要在
     frontend/src/utils/footprintGeo.ts。已有缓存包括
     normalizedProvinceCache、boundaryCache，组件内也已有一部分“初始化一次后 show/hide”的模式，可直接复用扩展。

     Recommended approach

     1. 调整国家名称标签显示阈值

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 将国家标签显示条件从当前默认缩放附近进一步下调，只在比默认更远的缩放级别才显示。
     - 保持默认进入地图时不出现其他国家名称；只有用户继续缩小地图时，renderCityNameLabels 中的国家标签分支才生效。
     - 复用现有 COUNTRY_LABEL_MAX_ZOOM / shouldShowCountryLabels() 机制，不新增额外状态。

     2. 消除缩放时的重复渲染触发

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 移除 adjustMapZoom() 中对 renderMapOverlays() 的直接调用。
     - 统一依赖现有 zoomend / moveend 事件来触发渲染，避免一次缩放同时触发“主动渲染 + 事件渲染”两遍。
     - 保留 dragstart / zoomstart 中对 hasUserMoved 的处理。

     3. 将多边形覆盖物改为“初始化一次，后续显隐/按需重建”

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 重点优化：
       - renderProvinceOutlineOverlays
       - renderWorldCountryBoundaryOverlays
       - renderVisitedCityOverlays
       - renderCityBoundaryOverlays
     - 策略：
       - 对仅随主题变化或数据变化才需要变动的覆盖物，首次创建后缓存实例；缩放时不再 clear + new。
       - 对只受缩放阈值影响的覆盖物，改为 show()/hide() 控制，而不是每次 remove/add。
       - 仅当 props.items、visitedProvinceNames 或 theme 变化时，才销毁并重建对应覆盖物。

     4. 区分“数据变更渲染”和“缩放渲染”

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 将当前统一的 renderMapOverlays() 拆成更明确的调度：
       - 一类处理静态/半静态覆盖物（世界边界、省界、足迹高亮区域）
       - 一类处理随缩放变化更频繁的内容（标签、marker 可见性）
     - 缩放时只更新必要层；数据变化时才全量同步。
     - 这样可以避免每次缩放都重建省界和足迹高亮区域。

     5. 优化标签与 marker 的更新方式

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - renderCityNameLabels()：避免每次缩放都 clearLabelOverlays() 后重新 new 全部 AMap.Text；优先改为按阈值
     show/hide，必要时才重建。

     Context

     用户希望首页足迹地图满足两个要求：
     1. 默认缩放下不显示其他国家名称标签，只有继续缩小到更远视角时才逐步显示。
     2. 当前每次缩小时都会重新绘制边界线和足迹高亮区域，导致交互明显卡顿；需要优化为缩放时尽量复用已有覆盖物，而不是重复销毁
     和重建。

     当前实现集中在 frontend/src/components/web/HomeFootprintMap.vue，地理数据缓存主要在
     frontend/src/utils/footprintGeo.ts。已有缓存包括 normalizedProvinceCache、boundaryCache，组件内也已有一部分“初始化一次后
      show/hide”的模式，可直接复用扩展。

     Recommended approach

     1. 调整国家名称标签显示阈值

     文件：frontend/src/components/web/HomeFootprintMap.vue

     Context

     用户希望首页足迹地图满足两个要求：
     1. 默认缩放下不显示其他国家名称标签，只有继续缩小到更远视角时才逐步显示。
     2. 当前每次缩小时都会重新绘制边界线和足迹高亮区域，导致交互明显卡顿；需要优化为缩放时尽量复用已有覆盖物，而不是重复
     销毁和重建。

     当前实现集中在 frontend/src/components/web/HomeFootprintMap.vue，地理数据缓存主要在
     frontend/src/utils/footprintGeo.ts。已有缓存包括
     normalizedProvinceCache、boundaryCache，组件内也已有一部分“初始化一次后 show/hide”的模式，可直接复用扩展。

     Recommended approach

     1. 调整国家名称标签显示阈值

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 将国家标签显示条件从当前默认缩放附近进一步下调，只在比默认更远的缩放级别才显示。
     - 保持默认进入地图时不出现其他国家名称；只有用户继续缩小地图时，renderCityNameLabels 中的国家标签分支才生效。
     - 复用现有 COUNTRY_LABEL_MAX_ZOOM / shouldShowCountryLabels() 机制，不新增额外状态。

     2. 消除缩放时的重复渲染触发

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 移除 adjustMapZoom() 中对 renderMapOverlays() 的直接调用。
     - 统一依赖现有 zoomend / moveend 事件来触发渲染，避免一次缩放同时触发“主动渲染 + 事件渲染”两遍。
     - 保留 dragstart / zoomstart 中对 hasUserMoved 的处理。

     3. 将多边形覆盖物改为“初始化一次，后续显隐/按需重建”

     文件：frontend/src/components/web/HomeFootprintMap.vue
     - 重点优化：
       - renderProvinceOutlineOverlays
       - renderWorldCountryBoundaryOverlays
       - renderVisitedCityOverlays
       - renderCityBoundaryOverlays
     - 策略：
       - 对仅随主题变化或数据变化才需要变动的覆盖物，首次创建后缓存实例；缩放时不再 clear + new。

     文件：frontend/src/components/web/HomeFootprintMap.vue

       - 一类处理随缩放变化更频繁的内容（标签、marker 可见性）
     - 缩放时只更新必要层；数据变化时才全量同步。
